# 学习笔记
## 一、Intel HAXM 安装失败

###  问题
在创建 AVD（Android Virtual Device）后点击运行，模拟器一直无法打开。

###  思考
后面发现模拟器启动是要HAXM，但是我却一直安装不了HAXM，一开始那个勾选HAXM一直是无法勾选状态
在查阅资料时，发现 Google 为 AMD 处理器和部分 Intel 环境提供了替代方案：**AEHD（Android Emulator Hypervisor Driver）**。虽然我的是 Intel CPU，但 AEHD 也能作为 HAXM 的替代。
后面摸索在SDK Manager → SDK Tools，勾选 **Android Emulator Hypervisor Driver （AEHD）下载完成后，
进入 SDK 目录下的AEHD文件夹右键 `silent_install.bat`，选择 **以管理员身份运行**。 安装成功后重启模拟器，就能正常启动了。

## 二、模拟器运行导致电脑死机

### 问题
模拟器可以启动并运行 App，但运行几次后电脑突然卡死，只能强制重启。

### 思考
一开始我以为是内存不足，就一直增加电脑的虚拟内存，后面还是一直死机，这说明硬件资源不是瓶颈，问题可能出在软件冲突上。
最近我为了方便阅读菜单，安装了 **Android Studio的汉化包**。会不会是它导致的？
后面我删除了汉化包，重启 Android Studio，再次运行模拟器，果然没有死机情况了。
我这时明白了开发工具要尽量保持原版IDE。英文界面虽然初期有学习成本，但能避免很多诡异问题。

### 问题

### 思考
后面发现预览只是预览，要在模拟器上完美运行还要加约束。

## 四、app无限闪退

### 问题
app一直闪退

### 思考
后面把代码贴给ai看，我才知道Button btnToContact = findViewById(R.id.button);这串代码要放在setContentView(R.layout.activity_main);后面。
否则程序就会一直崩溃，原因因为 setContentView 还没执行，布局文件还没加载，所以 R.id.button 这个控件在内存中还不存在。
findViewById 找不到按钮，就会返回 null。
